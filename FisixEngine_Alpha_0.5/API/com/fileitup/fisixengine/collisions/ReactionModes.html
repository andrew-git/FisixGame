Packagecom.fileitup.fisixengine.collisions
Classpublic final class ReactionModes

Holds constants used to set Collision Reaction Modes.
Collision Reaction Modes:


NONE
This is the simplest and fastest collision reaction mode. Use it whenever possible for faster simulations
This mode constitutes no friction and no bounce, and should be used for objects who's physical accuracy is not very important.

PHYSICAL
Takes into account the object's material and updates its velocity after a collision. Although this method is more accurate than NONE, it is still not completely physically accurate as this engine is built for games which usually do onot require great accuracy as long as it looks good.


Example
The following example demonstrates how to set an object's collision reaction mode 
  import com.fileitup.fisixengine.collisions.  import com.fileitup.fisixengine.core.  import com.fileitup.fisixengine.particles.  
  var myEngine:FisixEngine = new FisixEngine()
  var p:CircleParticle = myEngine.newCircleParticle(100,100,10)
  
  p.setReactionMode(ReactionMode.PHYSICAL)
